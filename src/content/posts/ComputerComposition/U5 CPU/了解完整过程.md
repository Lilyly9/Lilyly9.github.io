# 了解完整过程

1. <span data-type="text" style="background-color: var(--b3-card-info-background); color: var(--b3-card-info-color);">保护断点和程序进程</span>

    - 断点：返回地址（由异常类型和发生异常时PC值决定）  
      eg:![image](assets/image-20251106094144-uzwz9dz.png)
    - <span data-type="text" style="color: var(--b3-font-color9);">保存地址（保护断点）</span>

      断点保存在 **PC** 中  
      保护断点时，要将 PC 内容送到栈或特定寄存器中保存

      1. 能够 **支持** 异常和中断的 **嵌套** 处理，将断点保存在 **栈** 中
      2. 如果系统 **不支持** 多重中断嵌套处理，则可以将断点保存在 **特定寄存器-异常程序计数器(EPC)** <span data-type="text" style="background-color: var(--b3-card-warning-background); color: var(--b3-card-warning-color);"> </span>中
    - <span data-type="text" style="color: var(--b3-font-color9);">保存状态（保护程序进程）</span>

      异常处理后需要返回原来被中断的程序 - 保存被中断时原程序的状态

      1. 正常时 **PSW(程序状态字)**  保存在 **PWSR(程序状态字寄存器)**
      2. 发生异常/中断时要保存到 **栈** 或 **寄存器** 中
      3. 等到异常/中断处理完之后又从栈/寄存器中取出来回到 **PSWR**
2. <span data-type="text" style="background-color: var(--b3-card-info-background); color: var(--b3-card-info-color);">识别异常/中断事件并转相应程序执行</span>

    - 识别是什么异常/中断

      1. 内部异常事件

          1. 软件(程序)：CPU设置一个 **异常状态寄存器(Cause)** ，记录异常原因  
              os使用 **异常查询程序**，按照一定顺序 **查询** 异常状态 **寄存器**
          2. 硬件：也可以硬件通过预设的规则得到 **异常类型**，根据类型号转到相应的异常处理**程序**执行
      2. 外部中断源 

          在每条指令执行完后，取下调指令之前查询是否有中断请求

切换模式

从用户模式切换到内核模式

关闭中断(防止嵌套)

跳转执行

根据异常类型跳转到对应的处理程序入口地址

开始执行异常处理程序

恢复返回

处理完成后恢复保存的现场

执行ERET指令返回原程序继续执行详细说明一下
